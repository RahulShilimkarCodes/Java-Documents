# 1. Introduction

Exception handling is equally important in API Automation as it is in UI Automation.

A well-designed API framework should:

- Detect failures quickly
- Capture meaningful logs
- Generate useful reports
- Preserve root cause
- Support debugging

Many automation engineers focus only on:

```java
response.then().statusCode(200);
```

However, production API frameworks must handle:

- Network failures
- Invalid JSON
- Authentication issues
- Timeouts
- SSL issues
- Serialization failures
- Deserialization failures

A senior automation engineer must understand all these scenarios.

# 4. JsonPathException

One of the most common REST Assured exceptions.

Occurs when JSON path is invalid.

Example:

```java
String name =
response.jsonPath()
.getString("data.user.name");
```

Actual response:

```json
{
  "employee": {
    "name": "Rahul"
  }
}
```

Result:

```text
JsonPathException
```

---

## Why It Happens

Wrong JSON path.

Expected:

```text
data.user.name
```

Actual:

```text
employee.name
```

---

## Prevention

Always inspect response.

```java
response.prettyPrint();
```

Validate JSON structure before extraction.

---

# 5. Serialization Exceptions

Serialization converts Java objects into JSON.

Example:

```java
Employee employee =
new Employee();

String json =
objectMapper.writeValueAsString(
employee
);
```

Failure during conversion causes:

```text
JsonProcessingException
```

or

```text
Serialization Exception
```

---

## Common Causes

```text
Invalid Object Structure

Unsupported Data Types

Circular References

Missing Getters
```

---

## Prevention

Ensure POJO classes are properly designed.

---

# 6. Deserialization Exceptions

Deserialization converts JSON into Java objects.

Example:

```java
Employee employee =
objectMapper.readValue(
json,
Employee.class
);
```

If JSON does not match POJO:

```text
Deserialization Exception
```

---

## Example

JSON:

```json
{
  "salary":"abc"
}
```

POJO:

```java
private int salary;
```

Result:

```text
Type Mismatch Exception
```

---

# 7. NullPointerException in API Frameworks

Very common.

Example:

```java
String token =
response.jsonPath()
.getString("token");

System.out.println(
token.length()
);
```

If token is null:

```text
NullPointerException
```

---

## Prevention

Validate before use.

```java
Assert.assertNotNull(token);
```

# 9. AssertionError

Most common testing failure.

Example:

```java

response.statusCode(),
200
);
```

Actual:

```text
500
```

Result:

```text
AssertionError
```

---

## Important

AssertionError is not a REST Assured exception.

It is generated by:

- TestNG
- JUnit

during validation.

# 10. SocketTimeoutException

Occurs when server takes too long to respond.

Example:

```text
Request Sent

↓

Server Hangs

↓

Timeout Reached

↓

Exception
```

Result:

```text
SocketTimeoutException
```

---

## Causes

```text
Slow Server

Heavy Database Query

Network Delay

Application Performance Issue
```

---

## Prevention

Configure proper timeout values.

```java
RestAssured.config =
RestAssuredConfig.config();
```

Occurs when SSL negotiation fails.

Example:

# 14. API Validation Exceptions

Many frameworks create custom exceptions.

Example:

```java
public class
ApiValidationException
extends RuntimeException {

   public ApiValidationException(
   String message){

      super(message);
   }
}
```

Usage:

```java
if(response.statusCode()!=200){

   throw new
   ApiValidationException(
   "Unexpected Status Code"
   );
}
```

# 16. Logging Strategy

Never use:

```java
System.out.println(
e.getMessage()
);
```

Use:

```java

"API Validation Failed",

);
```

Log should contain:

```text
Request URL

Request Body

Response Body

Status Code

Stack Trace
```

# 18. Retry Mechanisms

Temporary failures occur due to:

```text
Network Latency

Environment Issues

Server Load
```

Retry Example:

```java
for(int i=0;i<3;i++){

   try{

      executeRequest();

      break;

   }
   catch(Exception e){

   }
}
```

---

## Best Practice

Retry only temporary failures.

Never retry:

```text
Validation Failures

Business Logic Failures
```
